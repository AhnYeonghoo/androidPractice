코틀린
- 객체지향 패러다임 ( Object Oriented Programming)

객체지향
- 객체를 통해서 문제 또는 원하는 바를 해결한다.

객체
- 축구게임
    - A: 축구선수 객체, 심판 객체, 경기장 객체, 관중 객체
    - B: 사람 객체, 공 객체, 호루라기 객체, 경기장 객체
    - 이와 같이 객체를 설계하는 것은 프로그래머의 경험, 능력
    - A안이 좋은 지, B안이 좋은 지 구별하기 위해서는 지식이 아니라, 경험이 더 중요하다.
    - 객체를 어떻게 구성하는 지가 노하우이자 개발을 잘하는 방법
        - 객체지향을 잘하기 위해서는 경험이 필요하니 서두룰 필요가 없다.
        - 하루 아침에 해결될 수 없다.

- 객체지향을 이해하는데 있어서 가장 중요한 문법적인 요소는 Class
    - Class를 통해서 객체를 만들 수 있기 때문.

클래스의 요소
- 설명서 (해당 클래스를 통해 객체를 만드는 방법)
- 객체의 기능에 대한 설명

생성자
- 객체를 만드는 방법
- 주 생성자 (Primary Constructor)
    - 실습강의
- 부 생성자 (Secondary Constructor)
    - 실습강의
- init 블록 (Initialize, 초기)
    - 초기화 블록
    - 초기화 시에 필요한 작업을 하는 곳
    - 블록 -> { }
