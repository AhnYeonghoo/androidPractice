프로그래밍 언어란

1. 프로그래밍 언어란 컴퓨터와 의사소통 하기 위한 규칙
- 컴퓨터
    - 하드웨어 (CPU, RAM, GPU, SSD)
    - 소프트웨어
- 보통 일방적인 의사 소통 (사용자 -> 컴퓨터)
- 하지만 컴퓨터도 사용자에게 말을 한다

2. 프로그래밍 언어의 종류
- 기계어
    - 1001 0001 -> 누산기 0001 위치의 값을 저장해라
    - 사람이 거의 이해할 수 없는 언어
- 어셈블리어
    - LOAD Y -> 누산기에 Y의 값을 저장해라
    - 사람의 거의 이해할 수 없는 언어
- 고급언어 (우리가 사용하는 언어)
    - 사람이 이해할 수 있는 언어(C, C++, Java, Kotlin, Python, Swift)

3. 고급 언어의 종류 (패러다임)
- 절차지향 언어
    - 위에서부터 아래로 순차적으로 프로그램이 실행된다.
- 객체지향 언어 (우리가 사용할 언어)
    - 객체라는 것을 만들어서 문제를 해결한다.
    - 코틀린, 자바는 패러다임이 똑같기 때문에 문법도 굉장히 유사함.
- 함수형 언어 (코틀린도 가능 but 객체지향을 지향)
    - 함수라는 것을 통해서 문제를 해결한다.

4. 어떤 언어가 가장 좋은 언어일까?
- 한글의 경우 다른 문자에 비해서 좋은 문자라고 말할 수 있지만, 프로그래밍에서 가장 좋은 언어는 없다.
- 프로그래밍 언어는 보통 목적을 가지고 탄생하였기 때문에, 목적마다 가장 좋은 언어가 다르다.
- 프로그래밍 언어는 사용자의 기호는 따라서도 충분히 선호도가 다를 수 있다.

5. 어떤 언어를 먼저 배우는 것이 좋을까?
- 어떤 언어를 먼저 배워야 하는 지는 중요하지 않다.
- 언어가 가지고 있는 패러다임을 공부하는 것이 더 중요하다

6. 언어의 선택
- 개발하려는 종류에 따라서 언어의 선택이 강제되는 경우가 많다.
    - 안드로이드 -> 코틀린, 자바, C++
    - iOS -> Object C, swift

7. 기타
- 프로그래밍 언어는 중국어, 한국어, 영어와 같이 사람이 사용하는 언어와는 다르다.
    - 프로그래밍 언어도 개발하는데 있어서 도구일 뿐이지, 한 번 배워서 평생 사용하는 기술이 아님.
    - 내가 잘 하는 언어가 선택의 기준이 되어서는 안된다.

코틀린을 배울 때 코틀린 문법 + 객체지향(OOP)를 배워야 한다.
